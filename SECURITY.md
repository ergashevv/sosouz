# Security Policy

## Supported Versions

This project is under active development.
Security updates are applied to the latest version on the `main` branch.

## Reporting a Vulnerability

If you discover a security issue, please report it privately:

- Email: `contact@soso.uz`
- Subject: `[SECURITY] soso vulnerability report`

Please include:

- a clear description of the issue,
- affected components/endpoints,
- reproduction steps or proof of concept,
- potential impact,
- suggested mitigation (if available).

## Response Targets

- Initial acknowledgement: within 72 hours
- Triage decision: within 7 business days
- Fix timeline: depends on severity and complexity

## Disclosure Policy

- Do not disclose vulnerabilities publicly before a fix is released.
- Coordinated disclosure is preferred after remediation.
- We will credit responsible reporters (if desired) after resolution.

## Security Best Practices for Contributors

- Never commit `.env`, API keys, credentials, or tokens.
- Rotate keys immediately if accidental exposure occurs.
- Keep dependencies updated and reviewed.
